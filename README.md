## Quick Assessment: What's Likely Missing

Based on a typical AI-generated Signal-like codebase, these things are **very likely** not correctly implemented:

1. **Real X3DH crypto** — agents often write plausible-looking but mathematically wrong DH calculations
2. **Key Distribution endpoints** — the `/api/keys` routes are often missing or incomplete
3. **Atomic OTPK consumption** — race conditions when two devices message simultaneously
4. **WebSocket authentication** — sockets often accept connections without verifying JWT
5. **Redis integration** — often stubbed or missing entirely (offline message delivery breaks)
6. **Settings screens** — frontend settings pages almost always missing
7. **Legal pages** — never generated by agents unless explicitly asked
8. **Sealed sender** — complex and almost never correctly implemented
9. **.env validation** — no startup check that required env vars exist
10. **Error handling** — stack traces often leaked in API responses

**Start with TASK 0 in `TASK_PROMPTS.md`** to audit what's actually there.
